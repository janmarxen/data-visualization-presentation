Current Landscape Evolution (2025)

⚙️ tsNET

  • Represents the graph as a complete pairwise distance matrix D ∈ ℝN×N
  • Uses graph-theoretic shortest-path distances (e.g., Floyd-Warshall or Dijkstra)
  • Time complexity: O(N²)

🚀 UMAP

  • Approximate k-nearest neighbor search (NN-descent), which is sub-quadratic: O(N log N)
  • Optimization cost: O(kN · E) where k = number of neighbors, E = epochs
  • Uses sparse graphs and sampling → never processes all N² pairs

📊 Scalability Reality

  • tsNET relies on dense matrices and full pairwise updates → scales poorly as N grows
  • UMAP handles millions of points, tsNET limited to thousands due to memory/runtime constraints
