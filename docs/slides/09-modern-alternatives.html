Current Landscape Evolution (2025)

⚙️ tsNET

  • Uses complete pairwise distance matrix D ∈ ℝN×N O(N²)
  • Computes graph-theoretic shortest-path distances (e.g., Dijkstra) O(N²)
  • Scales poorly. Limited to thousands of points due to memory/runtime constraints

🚀 UMAP

  • Approximate k-nearest neighbor search (NN-descent), sub-quadratic: O(N log N)
  • Uses sparse graphs and sampling → never processes all N² pairs
  • Handles millions of points
